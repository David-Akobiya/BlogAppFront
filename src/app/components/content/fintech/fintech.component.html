Зачем нужны открытые API

Преимущества, которые получаем при развитии открытых API

Новые сервисы

Создают недисриминационный доступ финтех-компаниям в инфраструктуру банка для разработки принципиально новых сервисов для клиентов банка

Снижение издержек

Способствуют появлению единых стандартов обмена данными между участниками финансового рынка и упрощают интеграцию систем

Скорость разработки

Увеличивают скорость разработки новых банковских сервисов и их доставку до клиентов за счет устранения излиших бюрократических барьеров

Общее описание открытых API для бизнеса

Какие виды открытых API для бизнеса можно внедрить

OAuth для интеграции партнеров

Позволяет быстро и оперативно интегрировать white lable партнерские сервисы в экосистему банка, например, если надо провести быстрый пилот для действующей клиентской базы.

Выставление и загрузка счета на оплату

1. Банк-получателя предоставляет API для создания в интернет-банке получателя средств счета на оплату.
2. Банк-плательщика напраляет запрос через API в банк-получателя. 3. Банк-получателя обеспечивает создание счета на оплату в Интернет-банке и предлагает подтвердить списание.

Получение выписки по счету

Для интернет-магазинов. Такой сервис есть но он построен на базе интернет-банка и активно не развивается (0 клиентов).

Получение списка счетов

Необходиом для ИТ-компаний, интернет-магазинов и онлайн-бизнеса для автоматизации работы своего собсвтенного платежного приложения (например, если необходимо выдавать клиенту определенный номер счета в зависимости от того, за какую услугу платит клиент).

Выставление и загрузка счетов на оплату

Предлагаемый пошаговый план для внедрения API по выставлению счетов

1. Договориться с банками

Самый первый этап, который необходимо предпринять - это начать переговоры с банками, которые могут быть заинтересованы в развитии подобного сервиса. Переговоры можно начать на площадке АФТ или/или напрямую.

2. Разработать стандарт обмена данными

В рамках переговоров необходимо достигнуть соглашения по обмену данными, которое может быть оформлено как публичная декларация открытого  API и доступна всем другим участникам рынка по авторизованному доступу.

3. Оценить и реализовать доработки

Для реализации доработок необходимо оценить ресурсы разработчиков, аналитиков и сопровождения на реализацию возможных к реализации открытых API.

4. Подключать новые банки

Для подключения новых банков необходимо проводить совместный PR реализованных стандартов и развитие открытых API (например, возможность получить и оплатить счет при покупке товара или услуги в интернет-магазине.
